## 1. 30 saniyede proje resmi

**CogniVault Clinical** = Türkiye özel kliniklerine satılan **multi-tenant SaaS**.
Her klinik kendi markalı public bir hasta sayfası alır; hasta o sayfaya hiçbir
giriş yapmadan girer, KVKK onayı verir ve klinik-spesifik bir AI asistanıyla
**WhatsApp / web chat / telefon** üzerinden randevu alır.

**3 farklı kullanıcı, 3 ayrı mental space:**

| Kim | Nereye girer | Ne yapar |
|---|---|---|
| **Hasta** | `/c/<klinik-slug>` (public, anonim) | KVKK onay + AI sohbet + randevu |
| **Klinik Personeli** (operatör/sekreter) | `/operator/*` (login, role=operator) | Doktor inbox, shadow review, conversation detail |
| **Klinik Admini** (owner) | `/clinic/admin/*` **(henüz yok)** | Branding, hekim, persona, slot konfigürasyonu |

**Teknik yığın:** FastAPI + SQLAlchemy + Alembic + Pydantic v2 / React 18 + Vite +
TanStack Query + React Router + i18n. PostgreSQL (prod) / SQLite (dev). Auth: JWT
(bcrypt), patient için kısa-ömürlü token. AI: şu an OpenAI/Anthropic, **hedef
local stack: Qwen2.5-7B + faster-whisper + Coqui XTTS**.

**Mevcut olgunluk:** Erken Beta. 1 pilot klinikle canlıya çıkma kapısında ama
3 kritik eksik var (aşağıda).

### Git
- Hiçbir destructive komut (force push, reset --hard) ÇALIŞTIRMA
- Co-Authored-By: Claude... commit'lerin sonuna eklenebilir (gönüllü)
- Asla `git config` değiştirme
- Sadece `git add <specific>` — `git add .` yapma

### Şifre/Test
- Tüm seed kullanıcılar `demo123` ile
- Default klinik slug: `demo-klinik` (config'de `clinical_default_clinic_slug`)
